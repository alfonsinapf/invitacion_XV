Solución al Error "Forbidden" en API de Spotify

Estás experimentando un error 403 (Forbidden) al intentar agregar canciones a una playlist de Spotify desde tu aplicación web. Estos son los pasos para resolverlo:

Problema Principal: Permisos de Playlist Familiar

La playlist que intentas modificar pertenece a una cuenta familiar, no a la cuenta del desarrollador registrada en Spotify Developer Dashboard.

Spotify tiene restricciones específicas para playlists en cuentas familiares para evitar modificaciones no autorizadas.

Solución Paso a Paso

1 Verificar la Configuración en Spotify Developer Dashboard
  • Asegúrate de que la aplicación esté registrada correctamente
  • Verifica que los Redirect URIs estén configurados exactamente como la URL de tu aplicación
  • Confirma que estés usando el Client ID correcto
2 Obtener Permisos Explicitos del Propietario

El titular de la playlist debe:

  • Iniciar sesión en Spotify y autorizar explícitamente tu aplicación
  • Compartir la playlist contigo con permisos de colaborador
  • Verificar que la playlist no esté configurada como privada o restringida
3 Modificar el Código para Manejar Errores

Actualiza tu implementación de la API para manejar correctamente los errores de permisos:

" + "El propietario debe autorizar la aplicación y compartir la playlist."); } if (!response.ok) { const errorData = await response.json(); throw new Error(errorData.error?.message || "Error al añadir la canción"); } // Éxito - actualizar la UI showSuccessMessage("¡Canción añadida exitosamente!"); } catch (error) { console.error("Error al añadir la canción:", error); showAuthStatus(`Error: ${error.message}`, "error"); } } // Función para verificar permisos async function checkPlaylistPermissions() { try { const response = await fetch( `https://api.spotify.com/v1/playlists/${SPOTIFY_CONFIG.PLAYLIST_ID}`, { headers: { 'Authorization': `Bearer ${spotifyAccessToken}` } } ); if (response.status === 403) { return { hasAccess: false, message: "No tienes permisos para acceder a esta playlist" }; } if (!response.ok) { return { hasAccess: false, message: "Error al verificar permisos" }; } const playlistData = await response.json(); // Aquí puedes verificar adicionalmente si la playlist es colaborativa // o si el usuario actual tiene permisos de escritura return { hasAccess: true }; } catch (error) { return { hasAccess: false, message: "Error al verificar permisos: " + error.message }; } }
4 Solicitar los Scopes Correctos

Asegúrate de estar solicitando todos los scopes necesarios:

// Scopes necesarios para modificar playlists const SCOPES = [ "playlist-modify-public", "playlist-modify-private", "playlist-read-private", "playlist-read-collaborative" ].join(" "); // Incluir estos scopes en la URL de autenticación const authUrl = `https://accounts.spotify.com/authorize?client_id=${CLIENT_ID}&response_type=token&redirect_uri=${encodeURIComponent(REDIRECT_URI)}&scope=${encodeURIComponent(SCOPES)}&show_dialog=true`;

Alternativas si los Permisos no son Posibles

Si no puedes obtener los permisos necesarios para modificar la playlist directamente, considera estas alternativas:

A Crear una Nueva Playlist

Crea una nueva playlist en la cuenta del desarrollador y permite que los usuarios añadan canciones allí:

// Función para crear una nueva playlist async function createNewPlaylist(name, description) { const response = await fetch(`https://api.spotify.com/v1/me/playlists`, { method: 'POST', headers: { 'Authorization': `Bearer ${spotifyAccessToken}`, 'Content-Type': 'application/json' }, body: JSON.stringify({ name: name, description: description, public: true // o false según necesites }) }); if (!response.ok) { throw new Error("Error al crear la playlist"); } return await response.json(); }
